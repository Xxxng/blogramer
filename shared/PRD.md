# PRD (Product Requirements Document)

## 1. CAPABILITY (핵심 가치)
- **대상 사용자**: 다수의 블로그(Tistory, WordPress, Naver, Blogger)를 운영하며 매일 대량의 포스팅을 관리해야 하는 운영자.
- **핵심 가치**: 외부 서버 없이 로컬 데스크톱(Windows/Mac)에서 AI를 활용해 콘텐츠를 생성하고, 직접 검토(에디터)를 거쳐 여러 플랫폼에 예약 발행할 수 있는 통합 워크플로우 제공.

## 2. CONSTRAINTS (제약 사항 및 불변 규칙)
- **Local-First 보안**: 모든 API 키 및 개인 자산은 로컬 SQLite DB(`blogramer.db`)에만 저장하며 외부 전송을 엄격히 금지함.
- **스케줄링 오프라인 처리**: 앱 종료로 인해 예약 시간을 놓친 경우, `실패` 처리하지 않고 **다음 날 동일한 시간으로 자동 연기(Postpone +24h)** 함.
- **환경 의존성**: Windows 크로스 컴파일 및 배포 편의성을 위해 SQLite는 CGO 의존성이 없는 `github.com/glebarez/sqlite` 만을 사용함.

## 3. 기능적 요구사항 (Functional Requirements)

### 3.1 계정 및 설정 관리
- **다중 플랫폼 지원**: Tistory, WordPress, Naver, Blogger 계정의 CRUD 및 상태 관리.
- **인증 엔진**: OAuth2 및 App Password 등 플랫폼별 맞춤형 인증 로직 구현.
- **API 설정**: OpenAI API Key 관리 및 플랫폼별 개별 설정 저장.

### 3.2 AI 콘텐츠 생성 (MVP)
- **주제 추출**: 구글 트렌드 및 사용자 정의 카테고리 기반 키워드 매칭.
- **프롬프트 엔진**: MVP 단계에서는 SEO 최적화된 프롬프트를 내부에 고정하여 사용 (추후 고급 커스텀 지원).
- **이미지 생성**: OpenAI DALL-E 3를 연동하여 본문 삽화 및 썸네일 자동 생성.

### 3.3 검토 및 승인 (Editor UI)
- **초안 관리**: 생성된 글은 즉시 발행되지 않고 `draft` 상태로 로컬에 저장됨.
- **에디터 UI**: 사용자가 마크다운 서식을 확인하고 직접 수정할 수 있는 에디터 화면 제공.
- **승인 액션**: 에디터에서 사용자의 '즉시 발행' 또는 '예약' 버튼 클릭 시에만 발행 프로세스 진행.

### 3.4 발행 및 스케줄링
- **통합 퍼블리싱**: 선택한 하나 이상의 블로그 플랫폼으로 동시/순차 발행.
- **스케줄러 큐**: 예약된 발행 건을 관리하고, 앱 실행 중 정해진 시간에 자동 실행.
- **이력 관리**: 발행 성공 시 URL 저장, 실패 시 사유 기록 및 재시도 로직.

## 4. 비기능적 요구사항
- **UX/UI**: Tailwind CSS v4 기반의 다크 모드 우선 디자인 및 실시간 작업 진행률(Progress) 표시.
- **에러 핸들링**: 백엔드(Go)의 상세 에러를 프론트엔드(React)에서 사용자 친화적인 메시지로 변환.

## 5. NON-GOALS & OPEN QUESTIONS
- **Non-Goals**: SaaS 호스팅, 모바일 제어, MVP에서의 무료 이미지 API 연동.
- **Open Questions**: 에디터 라이브러리 최종 선정, DALL-E 비용 최적화를 위한 이미지 생성 토글 옵션 필요 여부.

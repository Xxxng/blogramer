# CPS (Context, Problem, Solution)

## 1. Context (상황 및 배경)
- **시장 배경**: 블로그 운영은 SEO(검색 엔진 최적화)와 콘텐츠 마케팅의 핵심이지만, 다수의 계정을 동시에 관리하고 매일 글을 작성하는 것은 운영자에게 막대한 부담입니다.
- **기존 솔루션의 한계**: 클라우드 기반 자동화 서비스는 월간 구독 비용이 높고, 개인의 민감한 API 키를 서버에 맡겨야 하는 보안 우려가 존재합니다.

## 2. Problem (문제 정의)
1. **운영 효율 저하**: 포스팅 작성, 플랫폼별 로그인, 이미지 최적화 등 반복 작업에 너무 많은 시간이 소요됨.
2. **콘텐츠 품질 유지의 어려움**: 매일 새로운 아이디어를 내고 SEO를 고려한 글쓰기를 직접 수행하는 데 한계가 있음.
3. **데이터 주권 및 비용**: API 키 등 개인 자산을 제3자에게 위탁하는 것에 대한 거부감과 지속적인 서버 유지비 발생.

## 3. Solution (해결책: Blogramer)
1. **로컬 통합 관리 (Security & Cost)**:
   - Wails를 활용한 데스크톱 앱으로 구현하여 사용자 PC에서 모든 작업이 이루어지게 함.
   - 서버 비용 없이 직접 API 연동을 수행하며, 민감 정보는 로컬 SQLite DB에 암호화하여 저장.
2. **AI 기반 콘텐츠 파이프라인 (Efficiency)**:
   - OpenAI/Claude API를 연동하여 주제 선정, 본문 작성, SEO 태그 생성 자동화.
   - 사용자는 "승인"만 하면 되도록 하는 반자동(Semi-Auto) 및 완전 자동 모드 지원.
3. **플랫폼 통합 퍼블리싱 (Convenience)**:
   - 단일 인터페이스에서 여러 블로그(Tistory, WordPress, Naver, Blogger)로 즉시 또는 예약 발행 가능.
   - 발행 이력 및 성과(URL)를 한눈에 볼 수 있는 대시보드 제공.

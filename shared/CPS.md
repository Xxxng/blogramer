# CPS (Context, Problem, Solution)

## Context (맥락)
- **프로젝트명:** Blogramer (블로그 대량 생산 프로그램)
- **기술 스택:** Wails (Go + React + TypeScript), SQLite (Local DB), OpenAI/Claude API
- **목표:** 사용자가 로컬 환경에서 안전하게 다수의 블로그 계정을 관리하고, AI를 활용해 고품질의 블로그 포스팅을 생성 및 자동 발행하는 통합 데스크톱 애플리케이션 구축.

## Problem (문제/필요성)
1. **시간 소모:** 다수의 블로그(티스토리, 워드프레스, 네이버 등)를 수동으로 운영하는 데 막대한 시간과 노력이 필요함.
2. **콘텐츠 생성의 한계:** 매일 양질의 글을 작성하는 데 아이디어 고갈 및 작성 피로감이 발생함.
3. **보안 및 비용 문제:** 외부 클라우드 서비스 이용 시 API 키 유출 위험이 있으며, 매달 발생하는 구독료(서버 유지비)가 부담될 수 있음.
4. **통합 관리 부재:** 각 플랫폼마다 접속하여 글을 쓰는 번거로움과 발행 이력을 한눈에 보기 어려운 불편함.

## Solution (해결책)
1. **로컬 기반 데스크톱 앱:** Wails를 사용하여 사용자 PC에서 직접 동작하게 함으로써 서버 비용을 없애고 API 키를 로컬 DB에 암호화하여 저장(보안 강화).
2. **AI 콘텐츠 파이프라인:** OpenAI/Claude API를 연동하여 주제 선정부터 본문 작성, SEO 최적화 태그 생성까지 자동화.
3. **멀티 플랫폼 퍼블리싱:** Tistory, WordPress, Naver Blog, Blogger 등의 API를 통합 인터페이스로 구현하여 한 번의 클릭으로 동시 발행 가능.
4. **자동 예약 시스템:** 로컬 스케줄러를 내장하여 사용자가 지정한 시간에 맞춰 순차적으로 발행되도록 구현.
